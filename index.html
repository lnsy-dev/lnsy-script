lnsy-script

Agentic Javascript Runtime

Download

Pre-built binaries are available on the GitHub releases page.

macOS & Linux — install system-wide

git clone git@github.com:lnsy-dev/lnsy-script.git
cd lnsy-script
cargo install --path .

Builds a release binary and places it in ~/.cargo/bin/lnsy-script, which is on $PATH for any standard Rust install. After that, lnsy-script works from anywhere:

lnsy-script          # start REPL
lnsy-script file.js  # run a script file

Update

git pull && cargo install --path . --force

Linux (x86_64) — pre-built binary

curl -L https://github.com/lnsy-dev/lnsy-script/releases/latest/download/lnsy-script-linux-x86_64 -o lnsy-script
chmod +x lnsy-script
sudo mv lnsy-script /usr/local/bin/

Overview

lnsy-script is a QuickJS-powered JavaScript runtime that ships transformer models, vector search, graph databases, and multi-threaded workers as first-class globals. All models are compiled into the binary — no external dependencies for inference.

Runtime
QuickJS engine via rquickjs. REPL + script mode. Full Promise support.
BERT / NLP
Embeddings, sentiment, extractive QA, named entity recognition.
KNN
Text classification over MiniLM embeddings. Persistent JSON storage.
TRM
Tiny Recursive Model. Neural text classifier with recursive H/L-cycle reasoning.
Agents
JS workers in isolated OS threads. Message-passing concurrency.
Databases
SQLite, vector store, and property graph — all persistent.
Fetch
Standards-compatible HTTP client backed by Rust reqwest.
cl
Run shell commands. Promise-based, streaming output, timeout and kill support.
Static Server
HTTPS file server in a background thread. Self-signed TLS, zero config.
Tools
LLM tool-calling registry. Whitelist-only execution with JSON Schema validation.
Rust Library
Embed the full JS runtime in any Rust project. All types and modules are public.

cl

Run shell commands from JavaScript. Returns a Promise that resolves with the full result, or rejects on timeout or command-not-found.

OptionDescription
cwdWorking directory
envExtra environment variables
timeoutMilliseconds before the process is killed
shellShell to use (default: $SHELL or /bin/sh)
stdinData piped to the process's stdin
onStatusCallback invoked per output chunk — receives { stream, chunk, elapsed, kill }

Basic usage

var result = await cl("git log --oneline -5");
console.log(result.stdout);   // captured output
console.log(result.success);  // true if exit code 0
console.log(result.code);     // numeric exit code
console.log(result.duration); // elapsed ms

Streaming output

await cl("ping -c 4 example.com", {
  onStatus: function(s) {
    // s.stream  → "stdout" or "stderr"
    // s.chunk   → data for this chunk
    // s.elapsed → ms since start
    // s.kill()  → terminate early
    console.log(s.chunk);
  }
});

Errors

// Command not found — rejects
cl("nonexistent").catch(function(e) { console.log(e.message); });

// Timeout — rejects; err.result has partial output
cl("sleep 60", { timeout: 500 }).catch(function(e) {
  console.log(e.result.stdout);
});

Non-zero exit codes do not reject — check result.code or result.success.

KNN

Text classification using k-nearest neighbors over MiniLM sentence embeddings (384-dimensional). Supports persistence to JSON.

MethodDescription
train(data)Train on [{text, label}]
trainText(texts, labels)Train on separate arrays
query(text, k?)Return k nearest [{label, score}]
classify(text, k?)Return majority-vote label
var knn = new KNN("model.json");

knn.train([
  { text: "I love this",    label: "positive" },
  { text: "Absolutely awful", label: "negative" }
]).then(function() {
  return knn.classify("This is wonderful");
}).then(function(label) {
  console.log(label);  // "positive"
});

TRM

Neural text classification using a Tiny Recursive Model with recursive H-cycle/L-cycle reasoning (~7M params). Trained from scratch on your labeled examples and persisted to a directory. Unlike KNN, TRM learns a neural network — each call to train() triggers a full retrain.

MethodDescription
train(data)Train on [{text, label}] — appends data and retrains
trainText(texts, labels)Train on separate arrays — appends and retrains
query(text, k?)Return top-k [{label, score}] with softmax probabilities
classify(text)Return the highest-probability label
var trm = new TRM("./model_dir/");

trm.train([
  { text: "I love this",    label: "positive" },
  { text: "Absolutely awful", label: "negative" }
]).then(function() {
  return trm.classify("This is wonderful");
}).then(function(label) {
  console.log(label);  // "positive"
});

BERT

Transformer inference via EmbeddingServer. All ONNX models are bundled in the binary.

MethodReturns
generateEmbedding(text)Float32Array (384-dim)
askQuestion(context, question)String — extracted answer span
getSentiment(text){label, score}
getNamedEntities(text)[{entity, word}]

Sentiment

var es = new EmbeddingServer();

es.getSentiment("I absolutely love this!").then(function(r) {
  console.log(r.label);  // "POSITIVE"
  console.log(r.score);  // 0.98
});

Question Answering

es.askQuestion(
  "The Eiffel Tower is located in Paris, France.",
  "Where is the Eiffel Tower?"
).then(function(answer) {
  console.log(answer);  // "Paris"
});

Named Entity Recognition

es.getNamedEntities("Apple Inc. is in Cupertino.").then(function(ents) {
  ents.forEach(function(e) {
    console.log(e.entity + ": " + e.word);
    // ORG: Apple Inc.
    // LOC: Cupertino
  });
});

Agents

Multi-threaded JavaScript workers. Each agent runs in its own OS thread with a full runtime — including access to KNN, EmbeddingServer, fetch, and databases.

// main.js
var agent = new Agent("worker.js");

agent.onmessage = function(event) {
  console.log(event.data);  // result from worker
};

agent.postMessage({ task: "classify", text: "great product" });

for (var i = 0; i < 500; i++) { __agentPoll(); }
// worker.js
var knn = new KNN("model.json");

self.onmessage = function(event) {
  knn.classify(event.data.text).then(function(label) {
    self.postMessage(label);
  });
};

Call __agentPoll() in the main thread to flush the message queue from all active workers.

Databases

SQLite — Database

Document store with full-text search (FTS5). Stores arbitrary JSON objects.

var db = new Database("store.db");

db.addItem({ name: "alice", city: "portland" }).then(function() {
  return db.find("city", "portland");
}).then(function(rows) {
  console.log(rows[0].name);  // "alice"
});

Vector Store — VectorDatabase

Store embeddings with metadata. Query by cosine, euclidean, or manhattan distance.

var vdb = new VectorDatabase("vecs.db");
var es  = new EmbeddingServer();

es.generateEmbedding("cat").then(function(vec) {
  return vdb.addItem(vec, { label: "cat" });
}).then(function() {
  return es.generateEmbedding("kitten");
}).then(function(q) {
  return vdb.query(q, 5, "cosine");
}).then(function(results) {
  console.log(results[0].metadata.label);  // "cat"
  console.log(results[0].score);           // ~0.95
});

Graph — GraphDatabase

Property graph with typed nodes and directed, labeled edges.

var g = new GraphDatabase("graph.db");

g.addNode({ name: "Alice" }).then(function(a) {
  return g.addNode({ name: "Bob" }).then(function(b) {
    return g.addEdge({ source: a, target: b, name: "knows" });
  });
}).then(function() {
  return g.findNode({ name: "Alice" });
}).then(function(alice) {
  return g.getConnectedNodes(alice.id);
}).then(function(neighbors) {
  console.log(neighbors[0].name);  // "Bob"
});

Fetch

Standards-compatible fetch backed by Rust's reqwest. Supports GET, POST, PUT, DELETE, and PATCH.

Property / MethodDescription
response.statusHTTP status code
response.oktrue if 200–299
response.text()Promise<String>
response.json()Promise<Object>
// GET
fetch("https://api.example.com/items")
  .then(function(r) { return r.json(); })
  .then(function(data) { console.log(data); });

// POST
fetch("https://api.example.com/items", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "lnsy-script" })
}).then(function(r) {
  console.log(r.status);  // 201
});

Tools

A registry and executor for LLM tool-calling. Register named functions with JSON Schema parameter definitions — nothing can be invoked unless it's been explicitly added.

MethodDescription
addTool(def)Register {name, description, parameters, handler}
removeTool(name)Remove a registered tool
hasTool(name)Returns true if registered
listTools()OpenAI-compatible tool schema array
call(name, args)Direct call by name
call(tool_call)Execute raw LLM tool_call object
callMany(tool_calls)Execute array of tool_calls

Registration

var toolbox = new Tools([{
  name: 'get_weather',
  description: 'Get current weather for a city',
  parameters: {
    type: 'object',
    properties: { city: { type: 'string' } },
    required: ['city']
  },
  handler: async function(args) {
    var r = await fetch('https://api.example.com/weather?city=' + args.city);
    return r.json();
  }
}]);

Direct call

toolbox.call('get_weather', { city: 'Portland' }).then(function(result) {
  console.log(result);
});

LLM tool_call format

// Pass the raw tool_call object from an LLM response
toolbox.call({
  id: 'call_abc123',
  type: 'function',
  function: { name: 'get_weather', arguments: '{"city":"Portland"}' }
}).then(function(r) {
  console.log(r.tool_call_id);  // 'call_abc123'
  console.log(r.result);        // weather data
});

// Batch — execute all tool_calls from an LLM response
toolbox.callMany(response.tool_calls).then(function(results) {
  results.forEach(function(r) {
    messages.push({
      role: 'tool',
      tool_call_id: r.tool_call_id,
      content: JSON.stringify(r.result)
    });
  });
});

When a handler throws at runtime, call() returns { tool_call_id, name, error: true, result: message } rather than rejecting — so the LLM can recover gracefully. Missing required parameters and unregistered tool names throw ToolValidationError and ToolNotFoundError respectively. Extra undeclared parameters are silently stripped before the handler is invoked.

Static Server

Serve a local directory over HTTPS from a background thread. Does not block the JS event loop. On first run a self-signed TLS certificate is generated and saved to ~/.lnsy/lnsy-static.crt — subsequent runs reuse it so you only need to trust it once.

new StaticServer("/path/to/www", 8443);
// → https://lnsy-static.local:8443

Useful for local development — spin up a server, then drive it with fetch or test it from a browser, all within a single script.

new StaticServer("./public", 8443);

fetch("https://lnsy-static.local:8443/data.json")
  .then(function(r) { return r.json(); })
  .then(function(data) { console.log(data); });

One-time setup

Add the hostname to /etc/hosts:

echo '127.0.0.1  lnsy-static.local' | sudo tee -a /etc/hosts

Then trust the certificate. The startup output will print the exact path and the right command for your OS.

macOS — add to System Keychain:

sudo security add-trusted-cert -d -r trustRoot \
  -k /Library/Keychains/System.keychain \
  ~/.lnsy/lnsy-static.crt

Linux — Chrome / Chromium:

mkdir -p $HOME/.pki/nssdb
certutil -d sql:$HOME/.pki/nssdb -A -t 'C,,' \
  -n 'lnsy-static.local' -i ~/.lnsy/lnsy-static.crt

Linux — Firefox:

certutil -d sql:$HOME/.mozilla/firefox/<PROFILE>.default-release \
  -A -t 'CT,,' -n 'lnsy-static.local' -i ~/.lnsy/lnsy-static.crt

Linux — system-wide (Ubuntu / Debian):

sudo cp ~/.lnsy/lnsy-static.crt /usr/local/share/ca-certificates/lnsy-static.crt
sudo update-ca-certificates

Restart your browser after trusting. The certificate persists across server restarts — this is a one-time step.

Rust Library

lnsy-script also ships as a lnsy_script library crate. Embed the full JS runtime — including all globals, databases, ML models, and agents — into any Rust project.

Add to Cargo.toml

[dependencies]
lnsy-script = { git = "https://github.com/lnsy-dev/lnsy-script" }

Embedding the JS runtime

setup_context registers all classes and globals on a QuickJS context:

use rquickjs::{Context, Runtime};
use rquickjs::loader::{FileResolver, ScriptLoader};
use lnsy_script::setup_context;

fn main() {
    let runtime = Runtime::new().unwrap();
    runtime.set_loader(
        FileResolver::default().with_path("."),
        ScriptLoader::default(),
    );
    let context = Context::full(&runtime).unwrap();

    context.with(|ctx| {
        setup_context(ctx).unwrap();
    });

    context.with(|ctx| {
        ctx.eval::<(), _>(r#"
            var db = new Database();
            db.addItem({ name: "hello" }).then(function(id) {
                console.log("inserted id:", id);
            });
        "#).unwrap();
    });

    // drain pending promises
    loop {
        match runtime.execute_pending_job() {
            Ok(true) => {}
            _ => break,
        }
    }
}
